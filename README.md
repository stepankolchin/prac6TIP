### Краткое описание, зачем ORM и чем помог GORM.

- `ORM (Object-Relational Mapping)` нужен, чтобы можно было не писать SQL-запросы вручную. Вместо этого работа с базой данных ведётся через обычные объекты в коде, что гораздо удобнее.

- `GORM` в этом сильно помогает. С его помощью можно легко описать структуру таблиц и их связи. Запросы к данным строятся простыми методами, что уменьшает количество ошибок. Также не нужно постоянно думать о правильном экранировании переменных — `GORM` делает это сам. В итоге код становится чище, а разработка — быстрее.
